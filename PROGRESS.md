- Windows 전용(`windows.h`, `system`, `Sleep`).

## 2. 빌드 & 실행 (중요)

데이터 경로가 `test/` **상대경로**라 **반드시 `game/` 폴더에서** 빌드·실행해야 합니다.

```bash
cd game
gcc -std=c11 -Wall -Wextra -o game.exe code.c   # 또는 Visual Studio
./game.exe
```

> ⚠️ 현재 작업 PC에는 컴파일러가 없어 **빌드 검증은 사용자 환경에서** 해야 합니다.
> 0~4단계는 컴파일 검증 전(사용자가 나중에 한 번에 검증). 각 단계는 중괄호/괄호 균형·grep·코드 리뷰로 수동 점검.

## 4. 완료 단계 기록

### 0단계 — 파일 경로 이식성 + 입출력 안전성
- `C:\test\` 하드코딩 12곳 → 상대경로 매크로 + `open_or_warn()`, 모든 `fopen` NULL 처리

### 1단계 — 제어 흐름 정상화
- 재귀 `main()` 제거 → 반환 신호(`COMBAT_*`) 전파, 단일 `main` 루프가 흐름 제어

### 2단계 — 저장 일원화 + 버그 수정
- `write_save_files()` 통합, `auto_save` 7→9필드 버그, `feof` 루프, `cleanup()` 누수 해소

### 3단계 — 입력 처리 공통화
- `read_int()` 도입, `scanf` 11곳 교체, 슬롯/스텟 범위 가드

### 4단계 — 중복 제거 + 상수화 + 미사용 정리 (이번 작업)
- **포션 로직 중복 제거**: 거의 동일하던 `potion_use`/`fight_pouse`(각 ~120줄)를 삭제하고
  `use_one_potion()` 헬퍼 + 단일 `potion_menu()`로 통합 (조우 중·전투 중 공용)
- **상점 중복 제거**: `potion_shop`의 3종 구매 블록을 `buy_potion()` 헬퍼로 통합
- **매직넘버 상수화**: 포션 회복량/가격을 `POTION_*_HEAL`/`POTION_*_PRICE` `#define`으로 분리
- **미사용(죽은) 코드 제거**: `finish` 전역, `user.item` 필드, `cheat()`(미호출) 삭제
- 결과: 약 1357줄 → **1130줄**(−224). 변경 파일: [game/code.c](game/code.c)

### 4단계 확인 방법(사용자 환경)
1. 전투 중/조우 중 포션 사용이 이전과 동일하게 동작하는지(소/중/대, 보유 없음/회복 불필요).
2. 물약상점 구매(골드 충분/부족)가 정상인지.
3. 정상 빌드되는지(특히 `cheat`/`fight_pouse`/`finish` 미참조).

## 6. 다음 단계 제안

**5단계: 파일 분리.** `code.c`를 역할별 모듈로 분리한다. 컴파일러가 없어 헤더/extern 오류를 잡기 어려우므로, 위험을 줄이기 위해 **공용 헤더(`game.h`)에 타입·상수·extern 전역·프로토타입을 모으고**, `.c`는 역할별로 나눠 모두 `game.h`를 include 하는 방식으로 진행한다. `Makefile`도 추가한다.
