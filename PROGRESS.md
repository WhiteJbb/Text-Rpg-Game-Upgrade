- Windows 전용(`windows.h`, `system`, `Sleep`).

## 2. 빌드 & 실행 (중요)

데이터 경로가 `test/` **상대경로**라 **반드시 `game/` 폴더에서** 빌드·실행해야 합니다.

```bash
cd game
gcc -std=c11 -Wall -Wextra -o game.exe code.c   # 또는 Visual Studio
./game.exe
```

> ⚠️ 현재 작업 PC에는 컴파일러(gcc/clang/MSVC)가 없어 **빌드 검증은 사용자 환경에서** 해야 합니다.
> 0~2단계 코드는 아직 컴파일 검증 전입니다(사용자가 나중에 한 번에 검증 예정). 각 단계는 중괄호/괄호 균형·grep·코드 리뷰로 수동 점검했습니다.

## 4. 완료 단계 기록

### 0단계 — 파일 경로 이식성 + 입출력 안전성
- `C:\test\...` 하드코딩 12곳 → 상대경로 매크로(`DATA_PATH` 등) + `open_or_warn()` 헬퍼
- 모든 `fopen` NULL 처리(저장본 없음→첫 실행, 콘텐츠 없음→`exit(1)`, 저장 실패→`return`)

### 1단계 — 제어 흐름 정상화
- 재귀 `main()`/`adventure()`/`fight()` 호출 제거 → 반환 신호(`COMBAT_CONTINUE`/`TO_MENU`/`ONGOING`) 전파
- `monster/fight/attack/hp_out/damage` int 반환, 불필요한 `reg/region` 제거; 단일 `main` 루프가 흐름 제어
- 동작 변화: 물약상점 돌아가기→상점선택, 조우 중 아이템→조우 프롬프트, 미구현 지역→안내 후 복귀

### 2단계 — 저장 일원화 + 버그 수정 (이번 작업)
- 4중 중복 파일쓰기를 `write_save_files()` 하나로 통합 (`save`/`inside_save`/`auto_save`/`clear`)
- **`auto_save` 필드 버그 수정**: `mp`/`maxmp` 누락(7필드)으로 기록하던 것을 9필드 정식 포맷으로 (재실행 시 저장 깨짐 해소)
- **`while(!feof)` 버그 수정**: `mob_load`/`magic_load`가 EOF 뒤 쓰레기 1개 더 읽던 문제 → `fscanf` 반환값으로 판단
- **메모리 누수 해소**: `cleanup()`로 `mob_data`/`mg_data` `free`, `main` 종료(8·9번) 시 호출
- `clear()`의 지역변수 shadow 버그 제거(전역을 직접 초기화)
- 변경 파일: [game/code.c](game/code.c)

### 2단계 확인 방법(사용자 환경)
1. 사냥으로 exp/gold 획득 후 종료 → 재실행 시 포션 수량/스텟이 깨지지 않고 그대로인지(핵심: auto_save 버그).
2. 9번 초기화 → 재실행 시 새 캐릭터 생성 흐름으로 진입하는지.
3. 정상 빌드되는지(특히 새 `read_int`/`write_save_files` 없음, 컴파일 경고).

## 5. 남은 알려진 문제 (다음 단계 후보, 우선순위순)

1. **🟡 입력 검증 부재** — `menu()` 외 다수 `scanf` 미검증 → 문자 입력 시 무한루프. (3단계)
2. **🟡 포션 로직 중복** — `potion_use` ≈ `fight_pouse` (~120줄 거의 동일). (4단계)
3. **🟢 매직넘버 산재** — 회복량 20/100/500, 가격 100/500/1000, 스폰확률 등. (4단계)
4. **🟢 미사용 자산** — `speed`/`magic`/`crit`/`mp` 전투 미반영, `user.item`·`cheat()`·`finish` 죽은 코드. (4·6단계)
5. **🟢 인코딩 불일치** — `monsters.txt`=CP949(나머지 UTF-8) → 몬스터 이름 깨질 위험. (별도)
6. **🟢 Windows 종속** — 크로스플랫폼 시 추상화 필요(우선순위 낮음, C 유지).

## 6. 다음 단계 제안
