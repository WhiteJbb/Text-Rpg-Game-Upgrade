# CLAUDE.md

## 프로젝트 개요

이 프로젝트는 고등학교 시절 C언어로 직접 구현한 콘솔 기반 Text RPG Game을 리팩토링하고 확장하는 프로젝트입니다.

목표는 기존 게임의 추억과 핵심 구조를 최대한 보존하면서, 코드 품질·가독성·확장성·게임성을 점진적으로 개선하는 것입니다.

이 프로젝트는 단순히 새 게임을 만드는 것이 아니라, 기존 C 프로젝트를 기반으로 현대적인 구조로 개선하는 리팩토링 프로젝트입니다.

---

## 핵심 목표

1. 기존 Text RPG의 콘솔 감성과 플레이 흐름을 유지한다.
2. 기존 기능을 가능한 한 보존하면서 코드 구조를 개선한다.
3. 한 번에 대규모로 갈아엎지 않고, 작은 단위로 점진적으로 리팩토링한다.
4. 캐릭터, 전투, 아이템, 상점, 맵, 저장/불러오기 같은 기능을 확장 가능한 구조로 정리한다.
5. 나중에 포트폴리오에 넣을 수 있도록 코드와 문서 품질을 개선한다.

---

## 작업 원칙

* 변경 전에 반드시 관련 파일을 먼저 읽고 현재 구조를 파악한다.
* 바로 코드를 수정하지 말고, 먼저 어떤 문제가 있고 어떤 방향으로 고칠지 간단히 설명한다.
* 한 번에 너무 많은 파일을 수정하지 않는다.
* 기존 기능이 깨질 가능성이 있으면 먼저 알려준다.
* 기존 게임 로직을 삭제하기보다는 함수 분리, 구조체화, 파일 분리 방식으로 개선한다.
* 코드 변경 후에는 실행 방법, 테스트 방법, 확인해야 할 부분을 알려준다.
* 의미 없는 대규모 재작성은 하지 않는다.

---

## 절대 하지 말 것

* 기존 C 프로젝트를 갑자기 Python, JavaScript, Unity, Unreal 등으로 바꾸지 않는다.
* 기존 게임의 핵심 감성이나 흐름을 없애지 않는다.
* 사용자 입력 방식, 전투 방식, 스토리 흐름을 임의로 완전히 바꾸지 않는다.
* 빌드가 불가능한 상태로 큰 변경을 남기지 않는다.
* 확인 없이 파일 이름, 폴더 구조, 함수명을 대규모로 변경하지 않는다.
* 기존 코드를 삭제하기 전에 왜 삭제해야 하는지 설명한다.

## 개발 방향

이 프로젝트는 다음 순서로 개선한다.

### 1단계: 기존 코드 분석

* 전체 파일 구조 파악
* main 함수 흐름 파악
* 전투, 이동, 인벤토리, 상점, 저장 기능 등 주요 기능 분류
* 중복 코드, 긴 함수, 전역 변수, 하드코딩된 값 정리 대상 찾기

  * `map.c / map.h`

  * Player
  * Monster
  * Item
  * Skill
  * Map
  * GameState

### 3단계: 게임성 개선

* 전투 시스템 개선
* 몬스터 종류 추가
* 아이템과 장비 시스템 정리

* 레벨업 밸런스 개선
* 랜덤 이벤트 추가
* 퀘스트 또는 스토리 진행 구조 추가

### 4단계: 사용성 개선

* 메뉴 출력 개선
* 입력 예외 처리 추가
* 잘못된 입력을 받아도 프로그램이 종료되지 않도록 처리
* 화면 클리어, 구분선, 상태창 등 콘솔 UI 개선
* 한글 출력 인코딩 문제 확인

### 5단계: 포트폴리오화

* README 작성
* 빌드 방법 정리
* 게임 소개 작성
* 주요 리팩토링 포인트 기록
* Before / After 구조 비교
* 기능 목록과 향후 개선 계획 정리

## 코드 스타일

* C언어 기반으로 유지한다.
* 가능하면 C11 기준으로 작성한다.
* 함수는 하나의 역할만 담당하도록 작게 나눈다.
* 함수명은 의미가 명확하게 작성한다.
* 전역 변수 사용은 최소화한다.
* 매직 넘버는 `#define` 또는 `const`로 분리한다.
* 구조체를 적극적으로 사용해 데이터를 묶는다.
* 헤더 파일에는 구조체 정의, 상수, 함수 선언을 정리한다.
* 주석은 복잡한 로직이나 게임 규칙 설명에만 사용한다.

## 리팩토링 방식

코드를 수정할 때는 다음 순서를 따른다.

1. 현재 코드의 문제점 설명
2. 변경 방향 제안
3. 작은 단위로 코드 수정
4. 수정한 파일 목록 설명
5. 실행 또는 테스트 방법 안내
6. 다음 개선 후보 제안

가능하면 한 번의 작업은 하나의 주제만 다룬다.

예시:

* 전투 함수 분리
* Player 구조체 도입
* 아이템 로직 분리
* 입력 처리 함수 공통화
* 저장/불러오기 기능 정리
* README 작성

## 빌드와 실행

프로젝트에 Makefile이 없다면 나중에 추가한다.

기본 빌드 예시:

```bash
gcc -std=c11 -Wall -Wextra -o game main.c
```

파일이 분리된 후에는 다음과 같은 형태를 목표로 한다.

```bash
gcc -std=c11 -Wall -Wextra -Iinclude src/*.c -o game
```

가능하면 최종적으로 Makefile을 추가한다.

```bash
make
./game
```

Windows 환경에서는 실행 파일이 다음과 같을 수 있다.

```bash
game.exe
```

## Claude가 응답할 때 지켜야 할 것

* 코드를 수정하기 전에 먼저 현재 구조를 요약한다.
* 변경 사항은 작고 명확한 단위로 제안한다.
* “전체를 새로 짜는 방식”보다 “기존 코드를 살리는 방식”을 우선한다.
* 포트폴리오 관점에서 어떤 개선 포인트가 생기는지도 함께 설명한다.
* 빌드 오류 가능성이 있으면 미리 알려준다.
* 수정 후에는 반드시 확인해야 할 실행 명령어를 알려준다.
