# 행성: 지구 (Text RPG Game)

고등학생 때 C언어를 처음 배우고 만든 텍스트 기반 콘솔 RPG 게임과, 그 코드를 **현대적인 구조로 점진적으로 리팩토링한** 프로젝트입니다.

> 원본의 추억과 플레이 흐름을 보존하면서, 코드 품질·가독성·확장성·게임성을 단계적으로 개선했습니다.
> 리팩토링 과정 전체 기록은 [PROGRESS.md](PROGRESS.md), 작업 지침은 [CLAUDE.md](CLAUDE.md)를 참고하세요.

---

## 게임 특징

- 레벨 시스템과 경험치 획득 (사망 시 레벨 페널티)
- **4개 탐험 지역 · 몬스터 16종** (지역별 권장 레벨, 각 지역 보스 포함)
- 턴제 몬스터 전투 — **공격 / 마법(5종) / 아이템**
- **치명타**(크리티컬)·**회피**(민첩)·**마법 피해**(지력) 가 실제로 전투에 반영
- HP 포션(소/중/대) 관리 + **물약 / 마법 / 장비 상점**(골드로 구매)
- **장비 시스템** — 무기·방어구 10종, 힘/방어/지력 보너스 (구매 시 자동 착용, 전투에 반영)
- 스텟 분배 시스템 (힘/민첩/방어/지력/크리티컬/운)
- 슬롯머신 미니게임 (운 스텟이 당첨 확률에 영향)
- 데이터 저장 / 불러오기 (없으면 새 캐릭터 자동 생성)

## 빌드 & 실행

> 데이터 파일이 실행 폴더 기준 `test/` **상대 경로**라, **반드시 `game/` 폴더에서** 빌드·실행해야 합니다.

```bash
cd game
make                 # game.exe 생성 (gcc / MinGW)
# 또는 직접 컴파일:
gcc -std=c11 -Wall -Wextra -finput-charset=UTF-8 -fexec-charset=UTF-8 main.c util.c save.c player.c battle.c item.c -o game.exe
./game.exe
```

- **한글 출력**: 소스·데이터 파일은 모두 UTF-8이며, 실행 시 `main()`에서 콘솔을 UTF-8로 설정(`SetConsoleOutputCP`)합니다. Windows 기본 콘솔 코드페이지가 949(CP949)여도 한글이 정상 출력됩니다.
- **Visual Studio**: `game/`의 모든 `.c`를 한 프로젝트에 추가해 빌드하고, **작업 디렉터리를 `game/`로** 설정하세요. 한글이 깨지면 프로젝트 속성 → C/C++ → 명령줄에 **`/utf-8`** 옵션을 추가하세요(UTF-8 소스를 올바르게 해석).
- 필요 데이터(이미 `game/test/`에 포함): `monsters.txt`, `magicspell.txt`(콘텐츠) / `data.txt`, `stat.txt`(저장, 없으면 자동 생성).
- **Linux / macOS**: 동일하게 `gcc -std=c11 -Wall -Wextra *.c -o game` 으로 빌드됩니다. 화면 지우기·일시정지·딜레이는 `#ifdef _WIN32`로 플랫폼별 추상화되어 있어 수정 없이 동작합니다.

## 프로젝트 구조

리팩토링 전에는 단일 `code.c`(약 1,300줄)였으나, 역할별 모듈로 분리했습니다.

| 파일 | 역할 |
|---|---|
| `game.h` | 공용 헤더 — 상수, 구조체, `extern` 전역, 전체 함수 프로토타입 |
| `main.c` | 전역 변수 정의, 메인 루프(`main`/`menu`), 슬롯머신, 도움말 |
| `util.c` | 파일 열기(`open_or_warn`), 안전한 입력(`read_int`), 정리(`cleanup`) |
| `save.c` | 저장/불러오기/초기화, 데이터·몬스터·마법 로드 |
| `player.c` | 레벨/경험치, 스텟 보기·분배 |
| `battle.c` | 탐험·전투(`monster`/`fight`/`attack`/`magic`/`damage`) |
| `item.c` | 포션 사용·드롭, 물약 상점 |

## 게임 플레이

### 메인 메뉴
`1.탐험  2.정보확인  3.아이템  4.상점  5.스텟설정  6.슬롯머신  7.이용가이드  8.종료  9.초기화`

### 탐험 지역 (각 지역 일반 몬스터 3종 + 보스 1종)
- **몬스터의 초원** (Lv.1~) — 슬라임 / 고블린 / 코볼트 / [BOSS]오우거
- **늑대의 굴** (Lv.5~) — 들개 / 늑대 / 알파늑대 / [BOSS]웨어울프
- **엘프의 숲** (Lv.15~) — 픽시 / 다크엘프 / 트렌트 / [BOSS]엘프여왕
- **뱀파이어의 성** (Lv.30~) — 박쥐 / 좀비 / 가고일 / [BOSS]뱀파이어로드

> 권장 레벨 미만이면 입장이 제한됩니다. 보스는 약 1% 확률로 등장합니다.

### 스텟 시스템
| 스텟 | 효과 |
|---|---|
| 힘 | 공격력 증가 |
| 민첩 | 적 공격 **회피 확률** |
| 방어 | 받는 피해 감소 |
| 지력 | **마법(파이어볼) 피해** 증가 |
| 크리티컬 | **치명타(2배 피해) 확률** |
| 운 | 슬롯머신 당첨 확률 |

## 리팩토링 (Before → After)

고등학교 때 코드를 **기능을 보존하면서** 7단계에 걸쳐 개선했습니다. (작업 브랜치: `refactor`)

| 항목 | Before (원본) | After (리팩토링) |
|---|---|---|
| 코드 구조 | 단일 `code.c` ~1,300줄 | `game.h` + 6개 모듈 |
| 파일 경로 | `C:\test\` 절대경로 하드코딩 | `test/` 상대경로 매크로 |
| 흐름 제어 | 메뉴 복귀를 **재귀 `main()` 호출**로 처리(콜스택 누적) | 반환값(신호) 전파 + 단일 `while` 루프 |
| 파일 입출력 | `fopen` 실패 미확인(크래시) | `open_or_warn()`로 NULL 처리 |
| 저장 로직 | 4곳 중복 + `auto_save` 필드 누락 버그 | `write_save_files()` 일원화(버그 수정) |
| 입력 처리 | `scanf` 미검증(문자 입력 시 무한루프) | `read_int()` 방어 + 범위 가드 |
| 중복 코드 | 포션/상점 로직 ~240줄 중복 | 헬퍼 함수로 통합 |
| 미사용 스탯 | `crit`/`speed`/`magic`/`mp` 미반영 | 전투에 실제 반영 + 파이어볼 구현 |
| 메모리 | `malloc` 후 `free` 없음 | `cleanup()`으로 해제 |
| 입력 안정성 | `feof` 루프 버그(쓰레기 1개 더 읽음) | `fscanf` 반환값 검사 |

### 단계별 작업
- **0단계** — 파일 경로 상대화, 모든 `fopen` NULL 안전 처리 (이식성·안정성)
- **1단계** — 재귀 `main()` 제거 → 반환 신호 기반 제어 흐름으로 정상화
- **2단계** — 저장 로직 일원화, `auto_save` 데이터 손상 버그·`feof` 버그·메모리 누수 수정
- **3단계** — `read_int()`로 입력 검증 공통화, 슬롯/스텟 범위 가드
- **4단계** — 포션/상점 중복 제거, 매직넘버 상수화, 죽은 코드 제거 (약 −224줄)
- **5단계** — 단일 파일을 역할별 모듈(.c/.h)로 분리 + `Makefile`
- **6단계** — 미사용 스탯(치명타/회피/마법·MP)을 전투에 반영, 파이어볼 구현
